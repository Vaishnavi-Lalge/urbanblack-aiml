# urbanblack-aiml

AI/ML modules for the **Urban Black** cab platform — fare optimisation, ride classification, and data pipeline utilities.

---

## Repository Structure

```
urbanblack-aiml/
├── config/
│   └── config.yaml                  # Shared config (peak hours, fare params)
├── PricingNRevenue/
│   ├── LongTripOpt/                 # Long-trip fare calculator & data loader
│   │   ├── fare_calculator.py       # Two-tier fare engine (SHORT / LONG trips)
│   │   ├── extract_pdf.py           # Extract ride table from a PDF source file
│   │   ├── load_rides_dataset.py    # Load cleaned CSV into PostgreSQL
│   │   ├── create_rides_table.sql   # PostgreSQL schema for rides_dataset

│       ├── models/                  # Saved artefacts (generated by train.py)
│       │   ├── km_classifier.pkl
│       │   ├── scaler.pkl
│       │   ├── X_test.npy
│       │   └── y_test.npy
│       ├── notebooks/
│       │   └── exploration.ipynb
│       └── src/
│           ├── preprocess.py        # Feature engineering (returns unfitted scaler)
```

## Module 1 — OpNopModel (Operational / Non-Operational Classifier)

Classifies each ride request as **Operational** (real-time dispatch) or **Non-Operational** (analytical / deferred), using an XGBoost model trained on ride features (distance, demand, surge, time-of-day, etc.).

### Workflow

```
data/km_classifier_dummy_dataset.csv
        │
        ▼
  src/preprocess.py  ─── returns X, y, unfitted scaler
        │
        ▼
  src/train.py       ─── split → fit scaler on X_train → SMOTE on X_train → train models
        │                → saves  models/km_classifier.pkl
        │                         models/scaler.pkl
        │                         models/X_test.npy
        │                         models/y_test.npy
        ▼
  src/evaluate.py    ─── loads saved model + test split → prints metrics + confusion matrix
        │
        ▼
  src/api.py         ─── FastAPI endpoint  POST /classify
```

### Run training

```bash
cd PricingNRevenue/OpNopModel
python src/train.py
```

Expected output:
```
Preprocessing done (scaler NOT yet fitted — fit on X_train in train.py)
Logistic Regression trained
Random Forest trained
XGBoost trained
Best model saved  -> models/km_classifier.pkl
Scaler saved      -> models/scaler.pkl
Test split saved  -> models/X_test.npy, y_test.npy
```

### Run evaluation

```bash
python src/evaluate.py
```

### Start the API

```bash
# From PricingNRevenue/OpNopModel/
uvicorn src.api:app --reload --host 0.0.0.0 --port 8000
```

API Endpoints:

| Method | Path       | Description               |
|--------|------------|---------------------------|
| GET    | /health    | Liveness check            |
| POST   | /classify  | Classify a ride request   |

Example request:

```bash
curl -X POST http://localhost:8000/classify \
  -H "Content-Type: application/json" \
  -d '{
    "status": 1, "driver_availability": 1, "demand_level": 2,
    "fare_INR": 350.0, "rideKm": 12.5, "durationMin": 28.0,
    "pickup_lat": 18.52, "pickup_lng": 73.85,
    "drop_lat": 18.60, "drop_lng": 73.90,
    "peak_hour_flag": 1, "demand_supply_ratio": 1.3,
    "surge_multiplier": 1.2, "hour": 8,
    "day_of_week": 1, "month": 3
  }'
```

Example response:
```json
{
  "prediction": 1,
  "label": "Operational",
  "message": "Real-time processing"
}
```

## Module 2 — LongTripOpt (Long-Trip Fare Calculator)

Computes fares for trips > 30 km using a two-tier slab system derived from real Urban Black production data.

### Fare Structure

| Trip Type     | Economy (Rs/slab) | Premium (Rs/slab) | Peak Multiplier |
|---------------|-------------------|-------------------|-----------------|
| standard_long | 17.72             | 22.27             | 1.198 (+19.8%)  |
| outskirts     | 18.12             | 22.23             | 1.198           |
| airport       | 26.94             | 33.41             | 1.204           |
| intercity     | 27.50             | 33.71             | 1.198           |

Additional surcharges: +5% night (22:00–06:00), +8% holiday. Slab size = 1.5 km.

### Run fare calculator smoke-test

```bash
cd PricingNRevenue/LongTripOpt
python fare_calculator.py
```

### Extract data from PDF

```bash
# Auto-discover first .pdf in the folder:
python extract_pdf.py

# Or specify path explicitly:
python extract_pdf.py --pdf "dataset - Sheet1.pdf" --out rides_raw.csv
```

### Load rides dataset into PostgreSQL

```bash
# 1. Create the schema
psql -U postgres -d urbanblack_ride -f create_rides_table.sql

# 2. Load data (batch INSERT)
python load_rides_dataset.py \
  --csv rides_dataset_clean.csv \
  --host localhost --port 5432 \
  --dbname urbanblack_ride \
  --user postgres --password root \
  --create-table

# 3. Or use fast COPY for large files
python load_rides_dataset.py \
  --csv rides_dataset_clean.csv \
  --use-copy
```

## Bug Fixes Applied

| File | Bug | Fix |
|------|-----|-----|
| `fare_calculator.py` | Intercity rate typo (`22.34/28.14` → should be `27.50/33.71`) | Corrected to match docstring and production data |
| `fare_calculator.py` | `_slab_id` called before its definition | Moved definition above `compute_fare` |
| `preprocess.py` | Data leakage — `StandardScaler` fitted on full dataset before split | Returns unfitted scaler; caller fits on `X_train` only |
| `preprocess.py` | Data leakage — SMOTE applied to full dataset before split | SMOTE removed from `preprocess()`; now in `train.py` on train split only |
| `train.py` | `from preprocess import preprocess` fails outside `src/` directory | Added `sys.path.insert(0, Path(__file__).parent)` |
| `evaluate.py` | Re-ran `preprocess()`, re-fitting a different scaler on test data | Now loads persisted `X_test.npy` / `y_test.npy` from `train.py` |
| `api.py` | `"../models/km_classifier.pkl"` breaks when not run from `src/` | Replaced with `Path(__file__).parent.parent / "models" / ...` |
| `extract_pdf.py` | Hardcoded Windows path `d:\urbanblack-aiml\...\dataset.pdf` | Accepts `--pdf` CLI argument; auto-discovers PDF if omitted |
| `requirements.txt` | Missing `psycopg2-binary`, `pdfplumber`, `pyyaml` | Added all dependencies with minimum version pins |

## Java Back-End Services

The `urbanblack-backend/` directory contains the Spring Boot microservices (API Gateway, Auth, Ride, Driver, Fleet, Payment, Wallet, Notification, KYC, Employee, Traffic). Each service has its own `pom.xml` and `Dockerfile`. To run all services:

```bash
cd urbanblack-backend
docker-compose up --build
```

Refer to `urbanblack-backend/README.md` for full back-end setup instructions.
